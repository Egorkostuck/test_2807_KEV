Описание приложения:
Логика приложения писалась согласно тз.

1. В поле Block length вводится ширина блока, после нажатия кнопки Save блок перенесится на отрезок.
   Условная длинна отрезка 1000 ед., 1ед = 1px.

2. Перенос блоков выполняетя слева направо.

3. При одинарном клике на блок, он и все его части подсвечиваются.
   При двойном - удаляются, при этом остальные блоки остаются на своих местах.
   При удалении все оставшиеся блоки остаются на своих местах.

4. после добавляения нового блока в отрезок с пробелами, поиск нового места будет осуществлятся согласно следующему алгоритму:
    a. Место равное ширине новому блоку
    b. Место большее новому блоку
    с. 1-ое пустое место слева направо. Происходит делениен нового блока на части в зависимости от пустых мест.
       Если пустого места больше нет, то остаток переносится в конец.

5. Поле Block color задает цвет блоку, если блок разбивается на части, то все части наследуют 1 цвет.

6. Кнопка Sort объядиняет части 1-го блока.

7. Также добавлены алерты с ошибками на ввод корректных данных в поле Block length и переполнения отрезка на n-ое кол-во единиц.

8. Отрисовка блоков идет на основе массива из объектов.
    Пример:
        [
            {
                "id": "169082792678047",
                "value": 123,
                "left": 0,
                "emptySpace": 0,
                "blockColor": "#58c7fa",
                "uid": "lkr7ai2kl689t1dye4b"
            },
            {
                "id": "169082792712799",
                "value": 123,
                "left": 123,
                "emptySpace": 0,
                "blockColor": "#58c7fa",
                "uid": "lkr7aic749q137vhi39"
            },
            {
                "id": "169082792729965",
                "value": 123,
                "left": 246,
                "emptySpace": 0,
                "blockColor": "#58c7fa",
                "uid": "lkr7aigzolg3xyaa68s"
            }
        ]

9. После действий удаления, добавления, объединения блоков происходит перерисовка отрезка.
